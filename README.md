# JAV Collection Manager

Browse and manage your 115 netdisk collection (`E:\115\云下载`) through a local web UI.  
Reads file data from **Everything**, fetches JAV info live from **javbus.com** via an embedded iframe.

**No server needed.** Just run the scanner once, then open `index.html` directly in your browser.

---

## Requirements

| Requirement | Notes |
|---|---|
| [Everything](https://www.voidtools.com/) | Must be running and have indexed your disk |
| Python 3.8+ | Standard library only — **no `pip install` needed** |

---

## One-time setup — enable Everything's HTTP server

1. Open Everything
2. **Tools → Options → HTTP Server**
3. ✅ **Enable HTTP Server** (default port: **80**)
4. Click OK

> If port 80 is taken, change it to e.g. `8080` and also update  
> `EVERYTHING_PORT = 80` at the top of `scan.py`.

---

## Daily use

**Step 1 — generate data** (run whenever your collection changes):

```
python scan.py
```

This creates `data.js` in the same folder.

**Step 2 — open the UI:**

Double-click `index.html` (or drag it into your browser).

That's it. No server, no `python serve.py`, no port numbers.

---

## How Javbus info works

When you click any item in the list, a full-screen overlay opens showing the **javbus.com** page for that bango — just like an embedded browser window. No parsing, no API keys, no proxy server needed.

- Click **✕ Close** (or press the back button) to dismiss the overlay
- Click **↗ New tab** to open javbus in its own browser tab
- The "🗑️ Mark for Deletion" button is in the detail panel behind the overlay

---

## Views

| View | What it shows |
|---|---|
| **Dashboard** | Total size · item counts · top-20 series by count and by GB |
| **Browse** | All directories sorted by size (largest = most worth deleting) |
| **Statistics** | Full series table, sortable by count or size; click a row → browse that series |
| **Non-JAV** | Directories with no recognisable bango |

**Mark for deletion** → marks items with a red border; **Export list** downloads a `.txt` with the full paths.

## Supported bango formats

| Format | Example |
|---|---|
| Standard | `MIDE-332`, `SNIS-001`, `CAWD-100` |
| No dash | `MIDE332`, `SSNI001`, `EKDV460` |
| Dot separator | `STARS.001` |
| FC2 | `FC2-PPV-1234567`, `FC2PPV3456789` |
| HEYZO | `HEYZO-2345`, `heyzo_hd_2345` |
| Numeric prefix | `300MAAN-456`, `200GANA-123`, `230ORECO-171` |
| 1Pondo / Caribbean | `1PONDO-101015-001`, `102720-001-carib` |
| Site-prefixed | `[Thz.la] MIDE-332`, `hhd800.com@MIDE-332`, `0ses23.com0MIDE-332` |

## Troubleshooting

| Problem | Fix |
|---|---|
| Page shows "data.js not found" | Run `python scan.py` first |
| "Cannot reach Everything" in scan.py | Enable HTTP Server in Everything (see setup above) |
| Javbus overlay shows nothing | Check internet connection; click ↗ New tab to open directly |
| Bango not detected | Check the Non-JAV view; rename the folder to include the bango |
| Wrong bango | Rename the folder to use the correct bango, then re-run scan.py |
| After adding new files | Re-run `python scan.py` and reload the page (F5) |

## Optional: live mode with serve.py

If you prefer not to re-run `scan.py` manually, you can use `serve.py` which queries Everything live on each page load:

```
python serve.py
```

Then open **http://localhost:5000**. This requires the Everything HTTP server to be running at all times but skips the scan step.
